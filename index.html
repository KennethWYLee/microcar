1 小時 Boot Camp

60 分鐘內，帶學生完成第一台能動、能判斷的 Raspberry Pi Pico 小車。

這是一堂偏「入門體驗 + 初階程式邏輯」的課，不追求一次教完所有功能。 第一堂課的重點是讓學生知道：按一個指令，車子會動；讀一個感測值，車子能做不同反應。

進入完整 Boot Camp 頁
Boot Camp Snapshot

課程定位

  • 對象：國中生(Junior High)為主，也適合國小高年級。
  • 形式：2 到 4 人一組，邊看邊操作。
  • 目標：完成小車基本移動與感測反應。
  • 延伸：循跡(Line Following)與藍牙(Bluetooth)都放在後續單元。
60-min Plan

建議節奏

0-10 分鐘 認識 Raspberry Pi Pico 小車、Raspberry Pi Pico 控制板(Control Board)、安全提醒。
10-25 分鐘 前進、後退、左轉、右轉、停止五個基本動作。
25-40 分鐘 看懂感測器(Sensor)讀值，練習 if / elif / else。
40-60 分鐘 把小車動作和感測條件接在一起，做出第一個互動效果。
Materials

老師課前準備

Pico 小車(Car) Pico 控制板(Board) 電池(Battery) 感測器(Sensor) 電腦或投影 範例程式(Code)

第一堂課盡量只準備最核心的零件，學生比較不會被額外元件分散注意力。

First Win

這堂課學生的成功感

  • 我知道 `forward()` 和 `stop()` 在做什麼。
  • 我看得懂 `if` 會讓車子做不同動作。
  • 我能改一個速度、方向或距離數值。

Learning Outcomes

建議學習目標

  • 學生能說出輸入(Input)、判斷(Condition)、輸出(Output)的基本概念。
  • 學生能操作至少 5 個小車基本動作：前進、後退、左轉、右轉、停止。
  • 學生能看懂一段簡單的 `if / elif / else` 範例。
  • 學生能描述感測器數值改變時，小車為什麼會做出不同反應。
Teaching Route

建議授課順序

  1. 先讓學生看到小車真的會動。
  2. 再介紹動作函式(Function)和方向。
  3. 接著讓學生看懂感測器數值。
  4. 最後才把動作和條件判斷接起來。
Do Not Rush

第一堂課先不要放進來

  • 藍牙遙控(Bluetooth Control)
  • 過多硬體接線細節
  • 太長的完整專案程式
  • 一次講完所有感測器種類
Assessment

簡單評量方式

  • 請學生口頭說明一段程式會讓車子怎麼動。
  • 請學生改一個參數，觀察結果是否改變。
  • 請學生回答「為什麼距離太近時要停下來？」

學生區(Student Zone)

這裡用比較簡單的方式，帶學生一步一步看懂小車程式。

學生區的重點是「先看圖、再看短程式、最後自己改一點點」。 不用一開始就背很多名詞，只要先知道車子怎麼收到指令、怎麼做反應。

進入完整學生頁
Student Path

學生進來後，就照這 4 步做

1. 先認識動作

先看前進、後退、左轉、右轉、停止，不急著記全部程式。

2. 再看短程式

先看 `forward()`、`turn_left()` 這些小積木，知道它們代表什麼。

3. 下載第一份程式

先下載 `小車鍵盤控制範例`，跟著老師或網站一步一步試。

4. 做第一個挑戰

再挑戰 `距離感測範例`，看看不同距離為什麼會有不同反應。

小車移動(Car Motion)

你只要先看懂這 4 個函式

這些函式(Function)是小車的基本動作積木。只要知道它們代表什麼，後面就比較容易讀懂整份程式。

def forward(speed=30):
    ...

def backward(speed=30):
    ...

def turn_left(speed=30):
    ...

def turn_right(speed=30):
    ...

感測邏輯(Sensor Logic)

先讀值，再做判斷

這段程式最重要的是順序：先量距離，再根據距離決定顏色或動作。這就是條件判斷(Condition)的概念。

dist = sensor.ping()

if dist < 10:
    sensor.rgb_all(utils.COLOR_RED)
elif dist < 20:
    sensor.rgb_all(utils.COLOR_BLUE)
else:
    sensor.rgb_all(utils.COLOR_GREEN)
