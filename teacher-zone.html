Teacher Support

BootCamp 教師支援模組

本頁是 `BootCamp` 主題下的教師支援模組。 它不是另一條課程，而是協助教師完成同一個主題的課前準備、示範順序、下載流程與內容邊界。

先下載教學建議 第一個示範程式 回到 BootCamp

課堂情境以教師示範與學生跟做為主，適合用於 1 小時入門課程的授課安排。

Quick Route

先走 BootCamp，再用這頁支援

  1. 先看 BootCamp 頁
  2. 先開學生講義頁
  3. 下載教學建議
  4. 本堂課先停在基本動作控制

Real Teaching Context

這些畫面呈現實際授課情境

本區收錄與教學主線相符的課堂照片，用以呈現授課現場的實際樣貌。

教師可先在電腦教室進行全班示範，再讓學生分組練習，這樣最適合 1 小時入門課的節奏。

學生操作時需要同時看 Thonny 視窗與桌面上的小車，因此建議每組保留清楚的桌面空間。

Teacher Console

這一頁負責支援 BootCamp 主題的教學節奏

教師支援模組的任務是協助老師把同一個主題教得更穩，而不是額外再開一條新的內容線。

Finish Line

本堂完成線

  • 學生知道如何把 Raspberry Pi Pico 小車接上電腦。
  • 學生知道 `W / A / S / D / X` 各自對應的動作。
  • 學生至少成功完成一次前進、停止、轉向。
Open First

老師先開什麼

  1. 先看 BootCamp 的授課流程。
  2. 再開 學生講義頁，確認學生看到的版本。
  3. 再看 1 小時授課建議。
  4. 最後示範 鍵盤控制程式。
Not Today

本堂課暫不納入

  • 藍牙(Bluetooth) 遙控。
  • 感測器邏輯與循跡。
  • 太多函式細節與硬體延伸。

Real Equipment Setup

BootCamp 更適合先看真實設備

第一堂課以連線與鍵盤控制為主，因此先看真實小車、桌面與教室配置，比直接講腳位更符合教學需求。

真實小車本體

備課時可先用這張照片向學生指出 Pico 板、感測器、輪子與連線位置，建立基本認識即可。

桌面操作配置

示範時只要確認電腦、Thonny、小車與連接線位置清楚，學生就能快速進入打字控制的環節。

教室配置參考

若在電腦教室授課，建議先投影示範，再讓學生分組操作，可有效降低第一堂課的混亂感。

Before Class

課前先做什麼

先確認課堂目標

  • 今天是入門課，不是完整專案課。
  • 先讓電腦和車子連起來，再教會學生用鍵盤控制基本動作。
  • 不要急著講藍牙(Bluetooth)、感測器或太多延伸功能。

優先開啟這三份教材

  1. 學生講義頁
  2. 1 小時授課建議
  3. 小車鍵盤控制範例

In Class

上課時怎麼帶比較順

Part 1

先讓電腦和車子成功連線

先讓學生看到電腦真的能把指令送到車子，這是整堂課最重要的第一步。

Part 2

再讓學生操作五個動作

先把前進、後退、左轉、右轉、停止操作熟，不需要同一堂課再帶更多功能。

Part 3

收在鍵盤控制就好

這堂 BootCamp 的完成線，就是學生可以用鍵盤控制車子，不必再加感測器與循跡。

After Class

BootCamp 完成後再延伸

給學生的小延伸

  • 把速度改快或改慢。
  • 設計一段自己的動作路線，例如前進、右轉、停止。
  • 讓學生先猜按鍵結果，再實際操作驗證。

後續才加入的主題

  • 循跡(Line Following)
  • 藍牙遙控(Bluetooth Control)
  • 更多感測器整合
