# 網頁設計進度紀錄

最後更新日期：2026-04-10

## 目前定位

這個網站目前改成 `主題優先(topic-first)` 的教學架構，而不是單純依照使用者身分分流。

也就是：

- `Boot Camp` 是主題主線
- `教師支援` 是該主題的教師模組
- `學生練習` 是該主題的學生模組

未來新增的主題，也會沿用同樣的模式。

## 目前主題路線

目前已確定的主題順序如下：

1. `1 小時 Boot Camp`
2. `感測器的操控`
3. `循跡入門`
4. `循跡進階`
5. `PIT 演算法討論`

另外可獨立加開一個主題：

- `電路板入門`

這樣的好處是：

- 首頁可以更像課程地圖
- 每個主題都能各自擁有清楚步驟、圖片、程式與下載資源
- 教師與學生頁面不會看起來像兩套分開的網站，而是同一主題下的支援頁

## 公開首頁內容政策

首頁只保留會直接幫助老師與學生理解課程的內容。

不再放上首頁的內容：

- 我與你之間的工作紀錄
- 版面整理過程的說明
- 「目前只留 4 區」這類內部設計決策文字
- 與主教學路線無關的旁支教材

這些資訊應改放在：

- `WEB_DESIGN_PROGRESS.md`
- 後續的設計備忘錄
- 內部工作紀錄檔

## 目前首頁結構

目前首頁仍以四個主要入口為核心：

1. `1 小時 Boot Camp`
2. `教師支援`
3. `學生練習`
4. `下載區(Downloads)`

但首頁內容已開始往「主題路線圖」調整，而不是只強調使用者身分。

## 這次調整重點

### 1. 首頁重新改成公開課程語氣

首頁 Hero 已改成真正對老師和學生說話的版本，主軸是：

- 先學小車控制
- 再學感測器判斷
- 最後才走向循跡(Line Following)

### 2. 拿掉內部設計訊息

先前首頁曾出現偏內部工作的說明，例如：

- 「現在首頁只保留 4 個最重要的區塊」
- 類似專案整理或進度回報的句子

這類內容已移除，不再放在公開網站。

### 3. 下載區改成小車初階教材

下載區現在只顯示和初階小車教學直接相關的資源，例如：

- 教師用 1 小時授課建議
- 小車鍵盤控制範例
- 馬達基本函式範例
- 距離感測與 RGB 範例
- 補充 PDF 與 Mango 套件

與主線無關的電扇簡報與其他旁支檔案，已移到 `archive/`，不再留在首頁使用的下載清單中。

### 4. 教師區與學生區加入明確路徑

為了避免使用者點進區塊後不知道下一步做什麼，目前已補上：

- 教師區的 `Teacher Path`
- 學生區的 `Student Path`

這兩塊的設計目的：

- 老師一進教師區就知道先看什麼、先下載哪份檔案
- 學生一進學生區就知道先看任務、再下載哪份程式、接著做哪個挑戰

### 5. 首頁延伸成四個完整頁

目前網站已從單一首頁，延伸成：

- `index.html`
- `bootcamp.html`
- `teacher-zone.html`
- `student-zone.html`
- `downloads.html`

現在點擊各區，不再只是跳到首頁中的 block，而是能進到對應的完整教學頁。

### 6. 延伸教材重新分區

目前已把不屬於主線的內容另外整理成延伸教材：

- `AmebaAI / AmebaNN`
  視為另外一台車的教材
- `Board & Library`
  視為電路板與套件相關資源

這樣目前這台 Pico 小車的初階課程，和其他車型 / 板子相關內容就不會混在一起。

### 7. 主線已改成第一堂課可完成的版本

目前首頁與主要分頁都改成更貼近第一堂課的內容：

- 先讓電腦連上車子
- 先學會 `Thonny`
- 先完成 `W / A / S / D / X`
- 先讓學生成功控制小車

### 8. 新增學生講義與 Thonny 圖解

目前已新增一頁獨立的 `student-handout.html`，用途是：

- 給老師投影
- 給學生課堂上直接看
- 給課後列印或複習

Boot Camp 與學生區也補上更清楚的 `Thonny` 操作圖解和常見問題排除。

## 目前內容取向

### 教師區

教師區目前偏專業授課版本，重點是：

- 學習目標
- 授課順序
- 第一堂課先不要放進來的內容
- 簡單評量方式

### 學生區

學生區目前偏國中入門，也盡量讓國小高年級可讀：

- 任務式引導
- 鍵盤控制圖示
- 感測器邏輯圖
- 短程式區塊
