20-step Route

學生請照著下面 20 個步驟，從上往下完成

本區是本堂課的正式操作流程。每一列就是一個 step，左邊看文字，右邊看對應圖片，再一路往下做。

STEP 1

先把小車放好

把小車放在平坦桌面或地面，避免一開始就撞出去。

STEP 2

確認桌面空間

確認電腦、滑鼠和小車都在自己看得到的位置，操作時不要被線材卡住。

STEP 3

接上 USB 線

把 USB 線接上 Raspberry Pi Pico 小車與電腦。

STEP 4

等老師確認連線成功

先等老師確認每一組都連線成功，再一起往下操作。

STEP 5

打開 Thonny

在電腦上打開 `Thonny`，準備開始今天的程式操作。

STEP 6

先看右下角解譯器

確認右下角是否為 `MicroPython (Raspberry Pi Pico)`。

STEP 7

如果不是，就跟老師一起切換

若解譯器不是 Pico，先跟著老師一起切換，不要自己亂按其他設定。

STEP 8

建立新的 Python 檔

按 `File > New`，建立一個新的 `.py` 檔案。

STEP 9

往下滑到完整程式區

找到本頁稍後的「完整程式」區塊。

STEP 10

按下複製完整程式

按下「複製完整程式」，把今天要用的程式先複製起來。

STEP 11

回到 Thonny 貼上

切回 Thonny，把整份程式貼到剛剛建立的新檔裡。

STEP 12

先存檔

按 `Ctrl + S`，將檔案存成 `keyboard_car.py`。

STEP 13

按下綠色 Run

按綠色 `Run` 執行程式，讓小車開始等待指令。

STEP 14

把滑鼠點到 Shell

將滑鼠點到下方 `Shell`，接下來所有指令都在這裡輸入。

STEP 15

先輸入 x

先輸入 `x` 後按 Enter，確認知道怎麼停車，這是最安全的開始。

STEP 16

再測試 w、a、d、s

依序輸入 `w`、`a`、`d`、`s` 測試小車前進、左轉、右轉與後退。

STEP 17

每次動完先停車

做完一個動作後，記得先按 `x` 停車，再換下一個指令。

STEP 18

完成 w -> x

先完成 `w -> x`，確認前進與停止都成功。

STEP 19

完成轉向挑戰

完成 `w -> a -> x` 或 `w -> d -> x`，練習方向控制。

STEP 20

說出 5 個按鍵的意思

最後能說出 `W / A / S / D / X` 各自代表的動作，就完成今天的課。

4-stage Overview

這 4 格是課程階段總覽，不是學生實際操作步驟

學生真正要照著做的是下一區的 `20 個步驟`。這一區只用來幫老師掌握 1 小時課程節奏。

直接跳到 20 步驟
01 0-10 分鐘

連線成功

先讓每一組都把電腦和小車接好，這是整堂課最重要的起點。

  • 老師做：示範 USB 線和 Thonny 的位置。
  • 學生做：確認自己的車有成功連上。
  • 完成線：知道等一下要在哪裡輸入指令。
02 10-20 分鐘

老師示範

先由教師示範輸入指令，建立按鍵與小車動作的對應關係。

  • 老師做：在 Thonny Shell 輸入 `w / a / s / d / x`。
  • 學生做：口頭說出每個字母控制什麼。
  • 完成線：知道 `x` 是停止鍵。
03 20-40 分鐘

輪流操作

每個學生都要真的摸到一次操作，不只是看老師演示。

  • 老師做：提醒每做完一個動作先停車。
  • 學生做：依序試 `w`、`a`、`s`、`d`、`x`。
  • 完成線：每個人都能讓車子成功動一次。
04 40-60 分鐘

完成任務

最後用一個簡單小挑戰收尾，讓學生有完成感。

  • 老師做：出一段簡單動作題。
  • 學生做：完成「前進、左轉、停止」或「前進、右轉、後退、停止」。
  • 完成線：能說出 `W / A / S / D / X` 對應的動作。

本堂操作原則

  • 一個按鍵，對應一個動作。
  • 先熟悉 `w` 與 `x`，即可完成最基本的前進與停止。
  • 操作完一個動作，最好先停車再換下一個動作。

Teaching Sequence

建議教學順序

課堂投影可依照以下順序進行，每一步聚焦一個重點，較有利於學生跟上操作。

Step A

先講操作，不先講理論

先完成操作示範，再回頭說明函式與指令，可提高學習動機與理解度。

最後才看按鍵對應

學生先理解小車動作後，再閱讀 `if cmd == "w"` 這段，較容易建立對應關係。

回完整程式

Step 1

先匯入模組，設定腳位

本段設定控制小車所需的腳位與模組。

from machine import Pin
import sys
import time

M1_A = Pin(12, Pin.OUT)
M1_B = Pin(13, Pin.OUT)
M2_A = Pin(10, Pin.OUT)
M2_B = Pin(11, Pin.OUT)
LED = Pin(25, Pin.OUT)

Step 2

先做出「停車」和「往前」

本段可先聚焦 `stop()` 與 `forward()`，理解函式如何控制小車動作。

def _set_motor(pin_a, pin_b, direction):
    if direction > 0:
        pin_a.value(1)
        pin_b.value(0)
    elif direction < 0:
        pin_a.value(0)
        pin_b.value(1)
    else:
        pin_a.value(0)
        pin_b.value(0)

def stop():
    _set_motor(M1_A, M1_B, 0)
    _set_motor(M2_A, M2_B, 0)

def forward():
    _set_motor(M1_A, M1_B, 1)
    _set_motor(M2_A, M2_B, 1)

Step 3

再補齊後退、左轉、右轉

本段補齊後退、左轉與右轉，構成小車的基本動作集合。

def backward():
    _set_motor(M1_A, M1_B, -1)
    _set_motor(M2_A, M2_B, -1)

def turn_left():
    _set_motor(M1_A, M1_B, -1)
    _set_motor(M2_A, M2_B, 1)

def turn_right():
    _set_motor(M1_A, M1_B, 1)
    _set_motor(M2_A, M2_B, -1)

Step 4

讀取鍵盤指令

本段負責接收 `Thonny Shell` 輸入的單一指令。

print("請在 Thonny Shell 輸入 w / a / s / d / x 後按 Enter")

while True:
    cmd = sys.stdin.read(1)
    if not cmd:
        continue

    cmd = cmd.strip().lower()
    if not cmd:
        continue

Step 5

把按鍵對應到動作

本段將每個按鍵對應到一個動作函式，是本堂課的核心控制邏輯。

    if cmd == "w":
        forward()
    elif cmd == "s":
        backward()
    elif cmd == "a":
        turn_left()
    elif cmd == "d":
        turn_right()
    else:
        stop()

Practice Order

實際操作時，建議這樣練

先從最安全、最容易成功的開始。不要一開始就連續按很多指令。

練習 1

先試 `x`

先知道怎麼停車，這樣等一下比較安心。

練習 2

再試 `w` 然後 `x`

先讓車子前進一下，再停下來。

練習 3

加入 `a` 和 `d`

讓車子左轉、右轉，熟悉方向感。

練習 4

最後試 `s`

讓車子後退，再記得停車。
